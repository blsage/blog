I always assumed Apple would hand over the confetti from iMessage as a SwiftUI component one day. They never did, and I’m not sure why.

Every other confetti library I tried looked wrong: too sparse, too neon, too obviously physics-engine. I wanted exactly one thing: a view that rains down the same confetti iMessage does, with no modifiers and nothing to configure.

So I went looking, found an old write-up with the right idea, and discovered the code no longer compiled. I read back through the current documentation to bring it up to date, then wrapped the whole thing in a single SwiftUI view: [UnionConfetti](https://github.com/unionst/union-confetti). On a real device it looks like this:

![UnionConfetti raining down when a button is tapped](/confetti/rain.gif#phone "The same effect, on iOS.")

## How to use

One boolean, one overlay:

```swift
import SwiftUI
import UnionConfetti

struct ContentView: View {
    @State var showConfetti = false

    var body: some View {
        Button("Rain on me") {
            showConfetti = true
        }
        .frame(maxWidth: .infinity, maxHeight: .infinity)
        .overlay(ConfettiView(isPresented: $showConfetti))
    }
}
```

That’s the whole API. Flip `showConfetti` to `true` and it rains; it resets itself when the last piece lands. Drop the overlay on top of anything worth celebrating, like a successful payment:

![Confetti firing on a successful Apple Pay payment](/confetti/in-app.gif#phone "Confetti on a completed payment.")

## That’s it

There’s room to grow (blurring the back layer, exposing a few customizations), but for a confetti view that just behaves like iMessage and gets out of the way, it does the job.
