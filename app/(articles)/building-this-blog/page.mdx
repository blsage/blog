## Where the idea came from

This site is based on two others.

The design is from [Benji Taylor’s site](https://benji.org). I don’t really know anything about design, and he knows everything. And I couldn’t afford a designer,[^cpi] but I certainly can follow a spec.

The architecture comes from [Guillermo Rauch’s blog](https://github.com/rauchg/blog). Everything about how this is set up comes from his blog: the open source GitHub repo, the MDX files that are their own routes, and the one line in a JSON registry driving the index. No CMS, no database, no frontmatter parser.

So the site is a combo of the two: <Highlight>Benji’s aesthetic on Guillermo’s skeleton</Highlight>.

## How it works

Each post lives at `app/(articles)/<slug>/page.mdx` and ships at the root path (this one is just `/building-this-blog`). There’s no CMS and no frontmatter parser, because a post is a file and the file *is* the route:

```mdx title="app/(articles)/hello/page.mdx"
import { Counter } from "./counter";

export const metadata = { title: "Hello" };

This is a whole post. The button below is a React component,
imported one line up and dropped straight into the prose:

<Counter />
```

A single `posts.json` registry holds each post’s slug, title, and date; the homepage index and the shared article layout both read from it, so the index and the page header can never drift out of sync. Code is highlighted at build time with [rehype-pretty-code](https://rehype-pretty.pages.dev), every element a post might reach for is already styled, and the whole thing prerenders to static HTML.

## Posts that do things

Because a post is a real React route, the component it imports is really running. Here’s the one from the file above, live:

That’s the key.[^counter] A post about a gesture can contain the gesture. A post about an animation can play it. No CMS decides which slice of the web I’m allowed to use. The whole thing is my escape hatch.

## The fun parts

I tried to get as many of the details right from Benji’s site, because they’re what make it feel nice.

For instance, the margin notes are my actual handwriting. I used [Calligraphr](https://www.calligraphr.com) to make a custom font from a photo in my notebook, and loaded it in with `next/font`.

![A spiral notebook page filled with a grid of hand-drawn letters, one glyph per cell](/building-this-blog/handwriting.jpg "How I created my own font for the margin notes.")

The bracket that pulls each note to the side is drawn with [rough.js](https://roughjs.com), by way of [rough-notation](https://github.com/rough-stuff/rough-notation). Instead of a clean SVG stroke it sketches a slightly wobbly line, redrawn a little differently each time, then animates it in when the note scrolls into view. The hand-drawn underlines and circles elsewhere on the site work the same way.

To get a nice <Highlight>highlighter</Highlight> effect, I combine several randomized gradients, padding, and tilts, all seeded by the text itself. So it pools naturally, and never changes between renders.

And lastly, there’s a <Gradient>letter-by-letter gradient</Gradient> that interpolates between any two colors I choose.

## Make it yours

This whole site is a reusable template, [taylor-made](https://github.com/blsage/blog). Everything personal lives in just three files:

<dl>
  <dt>site.config.ts</dt>
  <dd>Name, location, metadata</dd>
  <dt>app/bio.mdx</dt>
  <dd>The homepage bio</dd>
  <dt>app/posts.json</dt>
  <dd>The post registry</dd>
</dl>

And you don’t have to touch them by hand. Clone the repo, open it in Claude Code, and paste this:

<aside className="prompt" data-label="Try it">
Clone https://github.com/blsage/blog and make it mine: put my name and city in site.config.ts, rewrite app/bio.mdx about me, empty app/posts.json, and start my first post at app/(articles)/hello/page.mdx.
</aside>

Paste that in and you’re ready to rock.
