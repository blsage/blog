I’ve wanted a place to write for a while. Not a Substack, not a Medium account — a small corner of the internet that I actually own, where a post can be anything a web page can be.

The bar for shipping it was low on purpose: if publishing a post takes more than writing markdown and pushing to git, I’ll never do it. So the whole system had to disappear behind the writing.

## Where the idea came from

Two sites shaped this one.

The look comes from [Benji Taylor’s site](https://benji.org). His pages are quiet in a way most personal sites aren’t — one narrow column, small type, dates tucked into the margins, and interactive demos that sit inline with the prose like they were always part of the paragraph. The first time I read one of his posts I scrolled back up just to look at it again.

The architecture comes from [Guillermo Rauch’s blog](https://github.com/rauchg/blog). His insight is that a blog doesn’t need a content layer at all: every post is just an MDX file that *is* its own route, plus one line in a JSON registry that drives the index. No CMS, no database, no frontmatter parser — the framework already knows how to render pages, so let posts be pages.

This site is the marriage of the two: Benji’s aesthetic on Guillermo’s skeleton.

## How it works

Each post lives at `app/(articles)/<slug>/page.mdx` and ships at the root path — this one is just `/building-this-blog`. The homepage index is driven by a hand-maintained registry:

```ts
export function getPosts(): Post[] {
  return [...postsData.posts].sort((a, b) => b.date.localeCompare(a.date));
}
```

Code blocks are highlighted at build time with [rehype-pretty-code](https://rehype-pretty.pages.dev), and the shared article layout reads the title and date for the current slug from the registry, so posts never repeat their own metadata. Everything prerenders to static HTML.

## Make it yours

The whole thing is a reusable template: [taylor-made](https://github.com/blsage/taylor-made). Everything personal lives in three files:

<dl>
  <dt>site.config.ts</dt>
  <dd>Name, location, metadata</dd>
  <dt>app/bio.mdx</dt>
  <dd>The homepage bio</dd>
  <dt>app/posts.json</dt>
  <dd>The post registry</dd>
</dl>

Swap those, add posts, and it’s yours.

[^1]: The counter above is an ordinary client component co-located in this post’s folder — no plugin system, no shortcodes.
