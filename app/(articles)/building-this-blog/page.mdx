I’ve wanted a place to write for a while. Not a Substack, not a Medium account: a small corner of the internet that I actually own, where a post can be anything a web page can be.

## Where the idea came from

Two sites shaped this one.

The look comes from [Benji Taylor’s site](https://benji.org). His pages are quiet in a way most personal sites aren’t: one narrow column, small type, dates tucked into the margins, and interactive demos that sit inline with the prose like they were always part of the paragraph. The first time I read one of his posts I scrolled back up just to look at it again.

The architecture comes from [Guillermo Rauch’s blog](https://github.com/rauchg/blog). His insight is that a blog doesn’t need a content layer at all: every post is just an MDX file that *is* its own route, plus one line in a JSON registry that drives the index. No CMS, no database, no frontmatter parser. The framework already knows how to render pages, so let posts be pages.

This site is the marriage of the two: <Highlight>Benji’s aesthetic on Guillermo’s skeleton</Highlight>.

## How it works

Each post lives at `app/(articles)/<slug>/page.mdx` and ships at the root path (this one is just `/building-this-blog`). There’s no CMS and no frontmatter parser, because a post is a file and the file *is* the route:

```mdx title="app/(articles)/hello/page.mdx"
import { Counter } from "./counter";

export const metadata = { title: "Hello" };

This is a whole post. The button below is a React component,
imported one line up and dropped straight into the prose:

<Counter />
```

A single `posts.json` registry holds each post’s slug, title, and date; the homepage index and the shared article layout both read from it, so a post never repeats its own metadata. Code is highlighted at build time with [rehype-pretty-code](https://rehype-pretty.pages.dev), every element a post might reach for is already styled, and the whole thing prerenders to static HTML.

## Posts that do things

Because a post is a real React route, the component it imports is really running. Here’s the one from the file above, live:

That’s the part I care about most.[^1] A post about a gesture can contain the gesture. A post about an animation can play it. The writing surface is the whole of the web, not a subset a CMS decided to allow.

## The fun parts

A few of the flourishes are worth opening up, since they’re what make a page feel handmade instead of generated.

The margin notes are set in my actual handwriting. I turned my own letters into a font, named with no imagination “Bens Font 3,” which tells you how many tries it took, and loaded it with `next/font`. So when a note leans out into the margin, that’s my hand, not a typeface doing an impression of one.

The bracket that pulls each note to the side is drawn with [rough.js](https://roughjs.com), by way of [rough-notation](https://github.com/rough-stuff/rough-notation). Instead of a clean SVG stroke it sketches a slightly wobbly line, redrawn a little differently each time, then animates it in when the note scrolls into view. The hand-drawn underlines and circles elsewhere on the site work the same way.

The <Highlight>highlighter</Highlight> isn’t a flat swatch of color. It’s a gradient that runs strong through the middle and thins toward the edges, with the padding, tilt, and opacity all nudged by a seed taken from the text itself. Every stroke pools where a real marker would, and yet it stays deterministic: the same words always get the same swipe, so nothing flickers between renders.

And the <Gradient>letter-by-letter gradient</Gradient> is exactly that. The text is split into characters, and each one is painted its own color, interpolated from the first letter to the last.

## Make it yours

This whole site is a reusable template, [taylor-made](https://github.com/blsage/taylor-made). Everything personal lives in just three files:

<dl>
  <dt>site.config.ts</dt>
  <dd>Name, location, metadata</dd>
  <dt>app/bio.mdx</dt>
  <dd>The homepage bio</dd>
  <dt>app/posts.json</dt>
  <dd>The post registry</dd>
</dl>

And you don’t have to touch them by hand. Clone the repo, open it in Claude Code, and paste this:

<aside className="prompt" data-label="Try it">
Clone https://github.com/blsage/taylor-made and make it mine: put my name and city in site.config.ts, rewrite app/bio.mdx about me, empty app/posts.json, and start my first post at app/(articles)/hello/page.mdx.
</aside>

That isn’t hypothetical: paste it in, and a minute later you’re holding your own copy of this site, named after you and ready to write in.
