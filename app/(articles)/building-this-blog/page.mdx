I’ve wanted a place to write for a while. Not a Substack, not a Medium account: a small corner of the internet that I actually own, where a post can be anything a web page can be.

<MarginNote note="still true">
  <p>The bar for shipping it was low on purpose: if publishing a post takes more than writing markdown and pushing to git, I’ll never do it. So the whole system had to disappear behind the writing.</p>
</MarginNote>

## Where the idea came from

Two sites shaped this one.

The look comes from [Benji Taylor’s site](https://benji.org). His pages are quiet in a way most personal sites aren’t: one narrow column, small type, dates tucked into the margins, and interactive demos that sit inline with the prose like they were always part of the paragraph. The first time I read one of his posts I scrolled back up just to look at it again.

The architecture comes from [Guillermo Rauch’s blog](https://github.com/rauchg/blog). His insight is that a blog doesn’t need a content layer at all: every post is just an MDX file that *is* its own route, plus one line in a JSON registry that drives the index. No CMS, no database, no frontmatter parser. The framework already knows how to render pages, so let posts be pages.

This site is the marriage of the two: <Highlight>Benji’s aesthetic on Guillermo’s skeleton</Highlight>.

## How it works

Each post lives at `app/(articles)/<slug>/page.mdx` and ships at the root path (this one is just `/building-this-blog`). The homepage index is driven by a hand-maintained registry:

```ts title="app/get-posts.ts"
export function getPosts(): Post[] {
  return [...postsData.posts].sort((a, b) => b.date.localeCompare(a.date));
}
```

Code blocks are highlighted at build time with [rehype-pretty-code](https://rehype-pretty.pages.dev), and the shared article layout reads the title and date for the current slug from the registry, so posts never repeat their own metadata. Everything prerenders to static HTML.

## The elements

Everything a post might need is styled and ready, in plain markdown or plain HTML. *Emphasis* sets in Newsreader italic, **strong** stays Inter, and inline `code` gets a quiet tint. Text can be <mark>marked</mark> like a highlighter, or run over with <Highlight>blue</Highlight> or <Highlight color="pink">pink</Highlight> ink that pools at the edges of the stroke. UI labels like <kbd>Continue</kbd> wear capsule outlines, and feature names can run through a <Gradient>letter-by-letter gradient</Gradient>.

None of it is special syntax. Markdown compiles to ordinary HTML, and each tag is styled exactly once: `*word*` becomes an `<em>`, `> line` a `<blockquote>`, a `#` heading an `<h2>`, and `[^1]` a small `<sup>` linked to a list at the foot of the page. The stylesheet quietly handles the rest (`<kbd>`, `<dl>`, `<hr>`, `<mark>`, `<figure>`), so the thing you actually write stays plain markdown and still arrives looking composed.

## Make it yours

This whole site is a reusable template, [taylor-made](https://github.com/blsage/blog). Everything personal lives in just three files:

<dl>
  <dt>site.config.ts</dt>
  <dd>Name, location, metadata</dd>
  <dt>app/bio.mdx</dt>
  <dd>The homepage bio</dd>
  <dt>app/posts.json</dt>
  <dd>The post registry</dd>
</dl>

And you don’t have to touch them by hand. Clone the repo, open it in Claude Code, and paste this:

<aside className="prompt" data-label="Try it">
Clone https://github.com/blsage/blog and make it mine: put my name and city in site.config.ts, rewrite app/bio.mdx about me, empty app/posts.json, and start my first post at app/(articles)/hello/page.mdx.
</aside>

That isn’t hypothetical: paste it in, and a minute later you’re holding your own copy of this site, named after you and ready to write in.
